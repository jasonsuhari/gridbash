# Fix selected pane scrolling

Date: 2026-07-10
Release target: unreleased

## Summary

- Restored mouse-wheel scrollback when the pane under the pointer is selected.

## What Changed

- Selected panes now keep wheel input inside GridBash scrollback even when the terminal app has enabled mouse reporting.
- Wheel events remain scoped to the pane under the pointer and are never broadcast to the selected set.

## Why It Matters

- Agent TUIs commonly enable terminal mouse reporting, which previously swallowed the wheel precisely during selected-pane workflows.

## Validation

- `cargo test --no-fail-fast`
- `cargo clippy --all-targets -- -D warnings`
- `npm test`
- Added a regression test with SGR mouse reporting enabled.

## Release Notes

- Selected panes can be scrolled normally without disturbing sibling panes.
