# Code of Conduct

GridBash should be a practical, focused, and respectful place to collaborate.

## Expected Behavior

- Be respectful of different experience levels, backgrounds, and viewpoints.
- Keep feedback specific to the work.
- Assume good intent while still being clear about bugs, risks, and tradeoffs.
- Accept responsibility when a mistake affects others.
- Help keep issues and pull requests actionable.

## Unacceptable Behavior

- Harassment, intimidation, threats, or sustained disruption.
- Personal attacks, insults, or derogatory comments.
- Publishing private information without permission.
- Sexualized language or imagery in project spaces.
- Repeated off-topic comments after maintainers ask for focus.
- Any conduct that would reasonably make participation unsafe or hostile.

## Scope

This code applies in all GridBash project spaces, including issues, pull requests, discussions, release channels, and direct maintainer communication about the project.

## Reporting

Report conduct concerns to the maintainer privately using the contact information available on @jasonsuhari's GitHub profile. Include links, screenshots, dates, and any other context that helps the maintainer understand what happened.

Do not use public issues for sensitive reports.

## Enforcement

Maintainers may remove comments, close or lock issues, block users, reject contributions, or take other action needed to protect the project community. Enforcement decisions will consider the impact on participants and the health of the project.
